Intruções Loop

📖 Introdução aos Laços de Repetição (Loopings)

Looping, ou instrução de repetição, permite repetir parte ou todo o código um número finito ou infinito de vezes. 💡 Um exemplo prático é o ato de estudar até entender um conteúdo, repetindo o processo até a compreensão.

🔄 Laço While: Funcionamento e Exemplos Práticos

O laço 'while' é utilizado quando o número de repetições é indeterminado. Ele testa uma condição e, se verdadeira, executa o bloco de código. A repetição continua até a condição se tornar falsa.

int contador = 1;

while (contador <= 5) {

System.out.println("Contador: " + contador);

contador++;

}

🛑 Atenção: Um loop while pode nunca ser executado caso a condição seja falsa desde o início.

🔍 Dica: Utilize um loop while para processos que dependem de uma condição dinâmica, como uma leitura de entrada do usuário.

🔁 Laço Do While: Diferenças em Relação ao While

O laço 'do while' garante que o código será executado pelo menos uma vez, pois a condição é verificada após o bloco de código.

do {

System.out.println("Executado ao menos uma vez");

} while (false);

⚖️ Diferença-chave:

while: testa a condição antes.

do while: executa ao menos uma vez, testando a condição depois.

🎯 Aplicação prática: Menus interativos, que precisam ser exibidos sempre ao menos uma vez.

🎨 Criando um Menu Interativo com Do While

import java.util.Scanner;

public class MenuInterativo {

public static void main(String[] args) {

Scanner scanner = new Scanner(System.in);

int opcao;

do {

System.out.println("\nMenu Interativo");

System.out.println("1 - Iniciar");

System.out.println("2 - Configurações");

System.out.println("3 - Sair");

System.out.print("Escolha uma opção: ");

opcao = scanner.nextInt();

switch (opcao) {

case 1 -> System.out.println("Iniciando...");

case 2 -> System.out.println("Abrindo configurações...");

case 3 -> System.out.println("Encerrando o programa...");

default -> System.out.println("Opção inválida!");

}

} while (opcao != 3);

scanner.close();

}

}

🔑 Por que usar o 'do while' neste caso?

O menu é mostrado ao usuário ao menos uma vez.

🧮 Laço For: Estrutura e Boas Práticas

O laço 'for' é ideal quando sabemos previamente o número de repetições.

💡 Estrutura básica:

for (inicialização; condição; atualização) {

    // Código a ser repetido
}

⚙️ Exemplo: Calcular a média de alunos

import java.util.Scanner;

public class MediaAlunos {

public static void main(String[] args) {

Scanner scanner = new Scanner(System.in);

int totalAlunos = 3;

double soma = 0;

for (int i = 1; i <= totalAlunos; i++) {

System.out.print("Digite a nota do aluno " + i + ": ");

double nota = scanner.nextDouble();

soma += nota;

}

double media = soma / totalAlunos;

System.out.printf("Média da turma: %.2f\n", media);

scanner.close();

}

}

🔍 Boas práticas:

Declare a variável de controle dentro do for sempre que possível.

Evite manipulações complexas dentro da instrução de atualização.

⚡ Loop Enhanced For (For-Each)

O enhanced for loop (ou for-each) é ideal para percorrer arrays e coleções sem precisar gerenciar índices manualmente.

int[] numeros = {1, 2, 3, 4, 5};

for (int num : numeros) {

System.out.println("Número: " + num);

}

🎯 Dica: O for-each não permite modificar o valor dos elementos diretamente.

🚦 Instruções de Controle: Break, Continue e Return

🛑 Break

Interrompe o loop imediatamente.

for (int i = 0; i < 10; i++) {

if (i == 5) {

break;

}

System.out.println(i);

}

Saída: 0 1 2 3 4

↪️ Continue

Ignora a iteração atual e avança para a próxima.

for (int i = 0; i < 10; i++) {

if (i % 2 == 0) {

continue;

}

System.out.println(i);

}

Saída: 1 3 5 7 9

🔙 Return

Finaliza a execução do método atual.

public class TesteReturn {

public static void main(String[] args) {

System.out.println(verificarParidade(4));

}

public static String verificarParidade(int numero) {

if (numero % 2 == 0) {

return "Par";

}

return "Ímpar";

}

}

🛠️ Erros Comuns e Boas Práticas

Loop Infinito 🚨

Modificação Inadequada da Variável de Controle ⚠️

Uso Desnecessário de break e continue ❌

🎬 Resumo Final

🔹 While: Quando o número de repetições é desconhecido.

🔹 Do While: Quando o bloco precisa ser executado ao menos uma vez.

🔹 For: Quando o número de repetições é conhecido.

🔹 Enhanced For: Para percorrer arrays de maneira simplificada.

🔹 Instruções de Controle: break, continue e return oferecem flexibilidade no fluxo de execução.
