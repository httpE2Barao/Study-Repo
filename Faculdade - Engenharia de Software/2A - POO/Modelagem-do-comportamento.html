Modelagem do comportamento

1. Diagrama de Atividade 🎯

"O fluxo do processo passo a passo"

O que é?

Ferramenta UML que representa o fluxo de trabalho de um sistema, mostrando a sequência de atividades e decisões. Ideal para processos de negócio e algoritmos.

Elementos Principais

Exemplo: Processamento de Pedidos 🛒

Início → Cliente realiza pedido.

Verificação de Estoque → Sistema confirma disponibilidade.

Pagamento → [Aprovado] → Embalagem ou [Negado] → Cancelamento.

Fluxo Paralelo → Embalagem + Verificação de Transporte.

Despacho → Pedido enviado.

2. Diagrama de Interação 🤝

"Combinação de fluxos e sequências"

Componentes Chave

Atividades: Etapas do processo (ex: "Análise de histórico financeiro").

Pontos de Decisão ❓: Losangos com condições (ex: "[Risco baixo?]").

Concorrência ⚡: Atividades paralelas (ex: verificação de dados em paralelo).

Exemplo: Empréstimo Bancário 🏦

Solicitação → Cliente envia pedido.

Análise Financeira → Sistema avalia crédito.

Avaliação de Risco → [Aprovado] → Contrato ou [Negado] → Rejeição.

3. Diagrama de Comunicação 📡

"Interações entre objetos sem ordem fixa"

Características

Objetos: Entidades que interagem (ex: Organizador de Eventos).

Mensagens: Troca de informações (ex: "Reservar salão").

Exemplo: Reserva de Eventos 🎪

Organizador → Solicita salão ao Gestor de Espaços.

Gestor de Espaços → Confirma disponibilidade.

Organizador → Solicita vagas ao Gestor de Estacionamento.

4. Diagrama de Tempo ⏳

"Mudanças de estado ao longo do tempo"

Elementos

Linha de Vida 📉: Duração de um objeto (ex: Protetor de Tela).

Estados 🔵: Condições do objeto (ex: "Ligado", "Desligado").

Exemplo: Protetor de Tela 🖥️

Estado Inicial: Desligado.

Após 2s sem atividade → Ligado.

Tecla pressionada → Volta para Desligado.
