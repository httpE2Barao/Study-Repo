Modificadores de Acesso

Programação Orientada a Objetos (POO) organiza código como entidades do mundo real. Em Java, tudo é representado por classes (modelos) e objetos (instâncias). Os modificadores de acesso controlam a visibilidade de atributos e métodos, garantindo encapsulamento e segurança! 🧰

🔒 Encapsulamento: Por que é importante?

Protege dados de acesso indevido.

Controla alterações com regras definidas.

Organiza o código limitando operações externas.

Exemplo de classe Pessoa:

public class Pessoa {

private String nome; // 🔒 Atributo privado

private int idade;

    // Construtor
    public Pessoa(String nome, int idade) {
        setNome(nome); // Usa método para validar
        setIdade(idade);
    }

    // Métodos públicos para acesso (getters/setters)
    public void setNome(String nome) {
        if (nome != null) this.nome = nome;
    }
}

📚 Modificadores de Acesso

1. 🌐 Public

Visibilidade: Ilimitada (classe, mesmo pacote, outros pacotes).

Uso Ideal: Métodos que precisam ser acessados globalmente.

Exemplo:

public class Calculadora {

public int somar(int a, int b) { // 🌐 Método público

return a + b;

}

}

2. 🔐 Private

Visibilidade: Apenas na própria classe.

Uso Ideal: Atributos e métodos internos (ex: calcularMedia()).

Exemplo:

public class Aluno {

private double[] notas; // 🔐 Atributo privado

private double calcularMedia() { // 🔐 Método interno

        // Lógica da média
    }
}

3. 🛡️ Protected

Visibilidade: Classe atual, subclasses e mesmo pacote.

Cuidado: Pode causar alto acoplamento. Prefira getters/setters!

Exemplo:

public class Veiculo {

protected String modelo; // 🛡️ Visível para subclasses

protected void iniciarMotor() {

        // Método acessível por herança
    }
}

4. 📦 Default (Pacote)

Visibilidade: Classes do mesmo pacote (sem modificador explícito).

Não recomendado: Falta de controle. Prefira private + getters públicos.

Exemplo:

class Produto { // 📦 Modificador default (sem "public")

int estoque; // Visível apenas no pacote

}

📌 Tabela Comparativa

💡 Boas Práticas

Atributos: Sempre use private + getters/setters públicos.

Métodos internos: Marque como private para ocultar implementação.

Herança: Evite protected excessivo. Use composição se possível.

Default: Evite! Priorize visibilidade explícita.
