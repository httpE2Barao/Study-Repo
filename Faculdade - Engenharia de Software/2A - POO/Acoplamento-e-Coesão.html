Acoplamento e Coesão

📚 Conceitos Fundamentais

🔗 O que é Acoplamento?

Mede a dependência entre componentes/classes

🚫 Alto acoplamento = Mudanças em um componente afetam outros

✅ Baixo acoplamento = Componentes independentes e flexíveis

🧩 O que é Coesão?

Grau de foco e unidade em um módulo/classe

🎯 Alta coesão = Métodos/atributos com objetivo único

🚫 Baixa coesão = Responsabilidades dispersas e confusas

⚖️ Relação Entre Acoplamento e Coesão

💡 Objetivo Ideal:

🛠️ Boas Práticas

🔧 Reduzir Acoplamento

🎭 Usar interfaces (contratos entre classes)

🔄 Princípio da Inversão de Dependência (DIP)

🚫 Evitar métodos estáticos/variáveis públicas

📦 Design modular (ex: microsserviços)

🎯 Aumentar Coesão

⭐ SRP (Single Responsibility Principle)

🔐 OCP (Open/Closed Principle)

✂️ ISP (Interface Segregation Principle)

📌 Agrupar funcionalidades relacionadas

💡 Exemplos Práticos

🏨 Caso 1: Classe GerenciadorHotel

// ❌ Anti-padrão (Baixa Coesão)
class GerenciadorHotel {
  void fazerReserva() { /*...*/ }
  void calcularConta() { /*...*/ }
  void enviarConfirmacao() { /*...*/ }
}

// ✅ Refatorado (Alta Coesão)
class GerenciadorReservas { /*...*/ }
class CalculadoraContas { /*...*/ }
class EnviadorConfirmacoes { /*...*/ }

💳 Caso 2: Processamento de Pagamentos

// ❌ Interface monolítica
interface ProcessadorPagamento {
  void cartao();
  void boleto();
  void pix();
}

// ✅ ISP aplicado
interface ProcessadorPix { /*...*/ }
interface ProcessadorCartao { /*...*/ }
