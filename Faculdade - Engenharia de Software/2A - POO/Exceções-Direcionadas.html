Exceções Direcionadas

Domine a criação e tratamento de erros específicos em suas aplicações!

📌 Introdução e Conceitos

⚡ O que são?

Exceções criadas por você para representar erros específicos do seu domínio, herdando de Exception ou subclasses.

🎯 Por que usar?

Representar problemas únicos da sua aplicação

Organizar erros em hierarquias claras

Facilitar tratamento granular de falhas

public class SaldoContaException extends Exception { // 🎯 Exemplo

    // Implementação personalizada
}

🔧 Implementação Passo a Passo

📥 1. Criando a Classe de Exceção

public class SaldoContaException extends Exception {

private float saldoConta;

    // 🛠️ Construtor com mensagem personalizada
    public SaldoContaException(float saldo) {
        super("Saldo excedeu o limite! Valor: " + saldo);
        this.saldoConta = saldo;
    }

public float getSaldoConta() { return saldoConta; }

}

🚨 2. Declarando e Lançando a Exceção

public float adicionaValor(float montante) throws SaldoContaException {

float total = saldoAtual + montante;

if (total > 20000) {

throw new SaldoContaException(total); // 💥 Lançando exceção

}

return total;

}

🛡️ 3. Tratando a Exceção

try {

float valorTotal = conta.adicionaValor(novoValor);

} catch (SaldoContaException e) {

System.out.println("Erro: " + e.getMessage());

    // 🏦 Lógica para criar depósito fixo
}

✅ Boas Práticas

🎯 Seja específico: Crie exceções para cada tipo de erro relevante

📚 Hierarquia: Organize exceções em classes pai/filho (ex: FinanceiroException → SaldoException)

📝 Mensagens claras: Descreva o erro de forma autoexplicativa

🧹 Não abuse: Use apenas para casos excepcionais, não para fluxo normal
