🗺️ Introdução aos Mapas em Java

Mapas são estruturas chave-valor (como dicionários), ideais para buscas eficientes por chaves.

Principais características:

Chaves únicas, valores podem ser duplicados.

Não indexados por posição (busca pela chave).

Implementações: HashMap, LinkedHashMap, TreeMap.

🔍 Tipos de Mapas e Aplicações

1. HashMap

Ordenação: Não ordenado.

Performance: ⚡ Mais rápido (tabela hash).

Thread-Safety: ❌ Não sincronizado.

Uso Ideal: Quando a ordem não importa e busca rápida é essencial.

Exemplo:

HashMap estoque = new HashMap<>();

estoque.put("Maçã", 50);

estoque.put("Banana", 30);

System.out.println(estoque.get("Maçã")); // 50

2. LinkedHashMap

Ordenação: Mantém ordem de inserção.

Performance: 🐢 Um pouco mais lento (lista encadeada + tabela hash).

Uso Ideal: Quando a ordem de inserção deve ser preservada.

Exemplo:

LinkedHashMap alunos = new LinkedHashMap<>();

alunos.put(1, "Maria");

alunos.put(2, "Pedro");

System.out.println(alunos); // {1=Maria, 2=Pedro}

3. TreeMap

Ordenação: Chaves ordenadas (crescente/personalizada).

Performance: 🐢 Mais lento (árvore rubro-negra).

Uso Ideal: Quando é necessário ordenação natural ou personalizada.

Exemplo:

TreeMap frutas = new TreeMap<>();

frutas.put("Laranja", 10);

frutas.put("Abacaxi", 5);

System.out.println(frutas); // {Abacaxi=5, Laranja=10}

⚙️ Classe Collections (Utilitários)

Métodos estáticos para manipulação de coleções:

Métodos Principais:

sort(List): Ordena uma lista (ordem natural ou via Comparator).

reverse(List): Inverte a ordem dos elementos.

shuffle(List): Embaralha elementos aleatoriamente.

frequency(Collection, Object): Conta ocorrências de um elemento.

Exemplo de Ordenação:

List nomes = new ArrayList<>(Arrays.asList("Zélia", "Ana", "Carlos"));

Collections.sort(nomes);

System.out.println(nomes); // [Ana, Carlos, Zélia]
