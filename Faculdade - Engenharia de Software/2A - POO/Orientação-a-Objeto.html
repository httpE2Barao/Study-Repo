Orientação a Objeto

🚀 Introdução à Orientação a Objetos (OO)

A Programação Orientada a Objetos (OO) surgiu como resposta à complexidade crescente no desenvolvimento de software. Esse paradigma organiza o código em torno de objetos que representam entidades do mundo real, promovendo:

✅ Reutilização de código

✅ Modularidade para facilitar a manutenção

✅ Escalabilidade para suportar crescimento

📌 Objetivos da OO:

Reduzir a complexidade 💡

Facilitar o reaproveitamento de código 🔄

Melhorar a organização e extensibilidade do software 🔧

🏗 2. Conceitos Fundamentais da OO

📌 Classe e Objeto

🔹 Classe: Define os atributos e métodos que os objetos terão.

🔹 Objeto: Uma instância de uma classe, representando uma entidade do mundo real.

📌 Os 4 Pilares da OO

🧠 Abstração → Oculta detalhes irrelevantes, focando no essencial.

🔐 Encapsulamento → Restringe o acesso direto aos dados do objeto.

📂 Herança → Permite que uma classe herde características de outra.

🎭 Polimorfismo → Permite a reutilização de métodos com comportamentos diferentes.

🎭 3. Abstração

A abstração simplifica sistemas complexos ao destacar apenas os detalhes essenciais.

💡 Exemplos:

🏢 Sistema de Estacionamento: O carro é representado apenas por atributos relevantes como placa, marca e modelo.

🐺 Sistema de Monitoramento de Animais: Um lobo-guará pode ser representado por atributos como nome, espécie e comportamentos (uivar, caçar, etc.).

✅ Benefício: Elimina informações desnecessárias, tornando o código mais organizado.

🔒 4. Encapsulamento

O encapsulamento protege os dados do objeto, permitindo que sejam acessados apenas por métodos específicos.

🔹 Modificadores de acesso:

Privado (-) → Apenas a própria classe pode acessar.

Público (+) → Qualquer parte do código pode acessar.

📌 Exemplo:

class Veiculo {

private String marca; // Só pode ser acessado dentro da classe

}

🛠️ Analogia: O encapsulamento funciona como uma "caixa preta" de um avião ✈️. O usuário interage com a interface, sem precisar conhecer os detalhes internos do funcionamento.

🏛 5. Herança

A herança permite que classes compartilhem características sem repetir código.

🔹 Superclasse (classe pai): Representa uma entidade genérica (ex.: "Veículo").

🔹 Subclasse (classe filha): Especializa a superclasse (ex.: "Carro" e "Moto").

📌 Exemplo:

class Veiculo {

String modelo;

}

class Carro extends Veiculo {

int numeroDePortas;

}

✅ Benefícios:

Evita duplicação de código (princípio DRY).

Centraliza alterações, garantindo consistência no sistema.

🎭 6. Polimorfismo

O polimorfismo permite que um método tenha diferentes implementações dependendo da classe.

📌 Exemplo:

class Veiculo {

void ligarMotor() {

System.out.println("Ligando motor genérico...");

}

}

class Carro extends Veiculo {

void ligarMotor() {

System.out.println("Ligando motor do carro...");

}

}

✅ Vantagem: Flexibilidade para adicionar novos comportamentos sem modificar a estrutura do código.

⏳ 7. Evolução das Linguagens de Programação

📜 Histórico:

Anos 60 🏛 → Linguagens imperativas (COBOL, Fortran).

Anos 70 🏗 → Modularização (Pascal, C).

Anos 80 📂 → Introdução da OO (Smalltalk, C++).

Atualmente 🚀 → Linguagens como Java, Python e TypeScript utilizam OO como base.

📌 A Orientação a Objetos revolucionou a forma como softwares são desenvolvidos!

🎯 9. Atividade Prática 4 – Aplicação dos Pilares da OO

📌 Objetivo:

✅ Implementar os conceitos de abstração, encapsulamento, herança e polimorfismo.

🔧 Ferramentas:

🖥 Astah UML (para diagramas).

💻 Android Studio (para implementação prática).

📍 Passos:

1️⃣ Criar Diagrama de Classes 📊

Modelar "Veículo", "Carro" e "Moto", destacando os conceitos da OO.

2️⃣ Responder Perguntas de Análise:

Encapsulamento: O acesso a um atributo privado gera erro? 🛑

Abstração: O método ligarMotor em itálico indica um método abstrato? 🤔

Polimorfismo: A implementação distinta de ligarMotor nas subclasses demonstra o conceito? 🎭

📌 Exemplo prático no código:

abstract class Veiculo {

abstract void ligarMotor();

}

class Carro extends Veiculo {

void ligarMotor() {

System.out.println("Ligando motor do carro...");

}

}

class Moto extends Veiculo {

void ligarMotor() {

System.out.println("Ligando motor da moto...");

}

}

✅ Conclusão: O código reflete os conceitos fundamentais de OO!
