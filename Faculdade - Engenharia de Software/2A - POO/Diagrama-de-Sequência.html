Diagrama de Sequência

🎬 Diagrama de Sequência UML: Guia Completo

🔍 Visão Geral

O Diagrama de Sequência é uma ferramenta essencial em UML para modelar interações dinâmicas entre objetos ao longo do tempo. Ele detalha como objetos trocam mensagens para executar processos, refinando casos de uso e oferecendo uma visão clara do fluxo de tarefas.

🎯 Importância e Objetivo

Objetivo: Visualizar a sequência de interações entre objetos em um sistema, garantindo clareza no fluxo de processos.

Complementaridade: Trabalha em conjunto com o Diagrama de Classes (estrutura estática) para representar o comportamento dinâmico do sistema.

🛠️ Elementos do Diagrama de Sequência

👤 Atores

Entidades externas (usuários, sistemas) que interagem com o sistema. Posicionados à esquerda do diagrama.

📦 Objetos

Representados por retângulos com nome da classe (ex: :SistemaBiblioteca).

Organizados da esquerda para a direita, no topo do diagrama.

⏳ Linha de Vida

Linha tracejada vertical abaixo do objeto, indicando sua duração no processo.

Ativação: Caixa retangular na linha de vida (período de execução de uma ação).

📨 Mensagens

Síncrona (seta sólida ➔): Remetente espera resposta.

Assíncrona (seta vazada ⇨): Remetente não espera resposta.

Criação/Exclusão: create (nova instância) ou X (fim da linha de vida).

🔄 Operadores de Controle

Opt: Ação opcional (ex: opt [condição]).

Loop: Repetição (ex: loop 3 vezes).

Par: Execução paralela.

Gate: Conexão com outros diagramas.

💻 Diagrama de Sequência no Astah

Principais Funcionalidades

✅ Diagramação Automática: Organiza elementos visualmente.

✅ Drag-and-Drop: Adiciona objetos facilmente.

✅ Integração com Diagrama de Classes: Vincula operações do diagrama de sequência a métodos de classes.

Passo a Passo no Astah

Criar Diagrama: Menu > Diagrama de Sequência.

Adicionar Objetos: Arraste classes/objetos para o diagrama.

Inserir Mensagens: Conecte objetos com setas (síncronas/assíncronas).

Configurar Operadores: Use Opt, Loop, etc., para fluxos complexos.

📚 Exemplo Prático: Sistema de Reserva de Livros

Componentes

Usuário: Solicita reserva.

Sistema de Biblioteca: Processa a requisição.

Catálogo de Livros: Verifica disponibilidade.

Banco de Dados: Armazena informações.

Sequência de Mensagens

Usuário → Interface: "Reservar livro X".

Interface → SistemaBiblioteca: Encaminha solicitação.

SistemaBiblioteca → Catálogo: Verifica disponibilidade.

Catálogo → BancoDeDados: Confirma reserva.

BancoDeDados → Catálogo: Retorna status.

SistemaBiblioteca → Usuário: "Reserva confirmada!".

@startuml

Usuário -> Interface: Reservar livro X

Interface -> SistemaBiblioteca: solicitarReserva()

SistemaBiblioteca -> Catálogo: verificarDisponibilidade()

Catálogo -> BancoDeDados: consultar()

BancoDeDados --> Catálogo: disponível

Catálogo --> SistemaBiblioteca: true

SistemaBiblioteca --> Usuário: "Reserva confirmada!"

@enduml
