🚀 Aplicação com Banco de Dados - Resumo Geral

🔎 1. Introdução ao NoSQL

NoSQL (🟣) refere-se a bancos de dados não relacionais, que diferem dos tradicionais bancos relacionais (🟢) por não utilizarem tabelas para armazenar dados. Em vez disso, os dados são mantidos em coleções que contêm documentos (geralmente no formato JSON), cada um deles organizado em pares chave-valor.

Exemplo: Imagine uma coleção chamada medicos, em que cada documento armazena informações como nome, telefone e especialidade, tudo em formato JSON.

:round_pushpin: Exemplos de Bancos de Dados NoSQL

MongoDB (🟢)

Redis (🔴)

Cassandra (🟡)

Amazon Dynamo (🟧)

O MongoDB se destaca por utilizar documentos em JSON, facilitando o armazenamento de dados no formato chave-valor.

🖥️ 2. Instalação e Configuração do MongoDB Compass Community

Faça o download da versão Community do MongoDB Compass (gratuita).

Instale conforme seu sistema operacional (Windows, macOS ou Linux).

Abra o MongoDB Compass e conectese ao seu servidor local (porta padrão: 27017).

Dica: Sempre verifique se o serviço do MongoDB está em execução antes de abrir o Compass.

🌐 3. Conexão com o Banco de Dados e Criação de Banco & Coleção

Na tela inicial do MongoDB Compass, clique em New Database.

Nomeie seu banco e crie uma nova coleção (p. ex.: medicos).

Após a criação, você já consegue inserir documentos em JSON diretamente por meio da interface.

Importe ou exporte dados em JSON usando as opções de import/export do MongoDB Compass.

🗂️ 4. Estrutura do MongoDB Compass e Importação de Dados JSON

O Compass exibe documentos JSON em um formato de tabela, mas permite visualizar cada registro em formato original.

É possível usar filtros para encontrar documentos específicos.

Você pode importar dados em massa diretamente no Compass, bastando selecionar o arquivo .json correspondente.

🛠️ 5. Introdução ao Express e Configuração do Projeto

O Express.js (🔵) é um framework para Node.js (🟢) que facilita a criação de aplicações web:

Gerencia rotas (GET, POST, PUT, DELETE).

Simplifica a adição de middlewares.

Facilita a resposta às requisições HTTP.

📑 6. Criando a Estrutura do Projeto e Configurando Rotas

Dentro do Express (🔵), criamos diferentes métodos:

GET: busca de dados

POST: inserção de dados

PUT: atualização de dados

DELETE: remoção de dados

Exemplo de rota simples:

// index.js
const express = require('express');
const app = express();

app.get('/', (req, res) => {

  res.send('Bem-vindo à MedApp!');
});

app.listen(3000, () => {
  console.log('Aplicação escutando na porta 3000');
});

Observação: Separe as rotas em arquivos dentro da pasta routes para manter o código organizado.

⚙️ 7. Introdução aos Bancos de Dados Relacionais e Não Relacionais

Relacionais (🟢): utilizam SQL e organizam dados em tabelas. Ex.: MySQL, PostgreSQL.

Não Relacionais (🟣): utilizam documentos e coleções, sendo mais flexíveis. Ex.: MongoDB.

🏗️ 8. Instalação e Configuração do Mongoose

O Mongoose (🟣) é um ODM (Object Data Modeling) para MongoDB. Ele:

Simplifica a interação com o banco de dados.

Fornece um sistema de Schemas e Models para validar e estruturar os dados.

Instalação:

npm install mongoose

Configuração:

// database.js
const mongoose = require('mongoose');

mongoose.connect('mongodb://localhost:27017/medDB', {
  useNewUrlParser: true,
  useUnifiedTopology: true
});

module.exports = mongoose;

📝 9. Criação de Modelos no Mongoose

Schemas definem a estrutura dos documentos e Models são a representação do schema para interagir com o banco.

// medicoModel.js
const mongoose = require('./database');
const Schema = mongoose.Schema;

const medicoSchema = new Schema({
  nome: String,
  especialidade: String,
  contato: String
});

const Medico = mongoose.model('Medico', medicoSchema);

module.exports = Medico;
2

Dica: Exporte o model e utilize-o em rotas, controllers ou services para realizar consultas, inserções e alterações.

🏗️ 10. Preparação do Banco de Dados Local

Por que usar um BD local?

Rapidez e controle total do ambiente de desenvolvimento.

Possibilidade de teste e aprendizado de forma mais livre, sem custos de hospedagem.

Uso do MongoDB Compass (interface gráfica) para gerenciar coleções e documentos.

Em projetos maiores, você pode migrar para o MongoDB Atlas (versão em nuvem) ou outro serviço de banco de dados gerenciado.

🌐 11. Criação da Aplicação com Express

O projeto Med App (💊) serve de exemplo para conectar:

Node.js + Express

Banco de dados MongoDB

Organização de pastas (models, services, routes, etc.)

Exemplo de inicialização:

// index.js
const express = require('express');
const app = express();

app.use(express.json());

// Rotas
app.get('/', (req, res) => {
  res.send('Bem-vindo à MedApp!');
});

app.listen(3000, () => {
  console.log('Servidor rodando na porta 3000!');
});

O próximo passo é definir rotas específicas e integrar o banco de dados via Mongoose.

🏢 12. Adição de Bancos de Dados na Aplicação

🔵 ORM (Sequelize) para Relacionais

Exemplo de uso do Sequelize (🟦):

const Sequelize = require('sequelize');
const sequelize = new Sequelize('database', 'username', 'password', {
  host: 'localhost',
  dialect: 'mysql' // ou 'postgres', etc.
});

🟣 ODM (Mongoose) para MongoDB

Exemplo de configuração do Mongoose (🟣):

const mongoose = require('mongoose');
mongoose.connect('mongodb://localhost:27017/medDB', {
  useNewUrlParser: true,
  useUnifiedTopology: true
});

Em NoSQL, não há tabelas, mas sim coleções; cada documento possui uma estrutura dinâmica, definida pelo seu Schema no Mongoose.
